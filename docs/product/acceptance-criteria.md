# MVP 验收标准

以下标准是第 1 至第 6 阶段自动化测试和最终演示的追踪源。

## UC-01 订单咨询

### AC-001

**Given** 已登录消费者拥有订单 A，**When** 从订单 A 发起咨询，**Then** API 返回 201 和会话 ID，数据库保存一个绑定订单 A、消费者和租户的活动会话。

### AC-002

**Given** 消费者不拥有订单 B，**When** 尝试从订单 B 发起咨询，**Then** API 返回资源不可用响应，且不创建会话、不泄露订单 B 的业务字段。

### AC-003

**Given** 同一消费者对订单 A 使用同一幂等键连续提交两次，**When** 两个请求完成，**Then** 返回相同会话 ID，数据库只有一个活动会话。

## UC-02 Mock Agent 回答

### AC-004

**Given** 会话为 BOT_ACTIVE 且订单待发货，**When** 消费者询问发货时间，**Then** Mock Agent 返回确定性待发货答复，消费者消息和答复各持久化一次。

### AC-005

**Given** 问题不匹配任何 Mock 规则，**When** 消费者发送问题，**Then** UI 显示无法自动解决的答复和可用的转人工操作，会话保持可继续交互。

### AC-006

**Given** 远程 Agent 超时，**When** AgentGateway 达到 15 秒超时，**Then** Java 返回降级答复、记录依赖失败和关联 ID，不丢失消费者消息。

## UC-03 转人工

### AC-007

**Given** BOT_ACTIVE 会话，**When** 消费者请求转人工，**Then** 会话变为 WAITING_HUMAN，创建一个活动队列项并发布转人工事件。

### AC-008

**Given** 会话已经 WAITING_HUMAN，**When** 再次请求转人工，**Then** API 返回现有队列状态，活动队列项数量仍为一。

### AC-009

**Given** CLOSED 会话，**When** 请求转人工，**Then** API 返回 409 和稳定业务错误码，不创建队列项。

## UC-04 客服接入

### AC-010

**Given** 两名有权限客服同时接入同一等待会话，**When** 并发请求完成，**Then** 只有一人获得会话，另一请求返回 409，当前分派保持唯一。

### AC-011

**Given** 无 conversation:accept 权限的员工，**When** 接入等待会话，**Then** API 返回 403，会话和队列项不变。

### AC-012

**Given** 客服已接入且 WebSocket 临时断开，**When** 客服发送回复并随后重连，**Then** 消息已持久化且消费者可按消息游标补拉到该回复。

## UC-05 质量售后申请

### AC-013

**Given** 本人订单项在售后期内且上传合法图片，**When** 提交质量问题，**Then** 创建 SUBMITTED 申请，返回申请编号并保存租户隔离的凭证引用。

### AC-014

**Given** 上传文件类型不允许或大小超限，**When** 提交申请，**Then** API 返回 400，不创建申请、不保留对象存储孤儿。

### AC-015

**Given** 同一订单项使用相同幂等键重复提交，**When** 两个请求完成，**Then** 返回同一申请，数据库只有一条有效申请。

## UC-06 创建工单

### AC-016

**Given** 客服正在处理相关会话且分类配置有效，**When** 确认工单草稿，**Then** 创建工单、首条处理记录和部门分派，并发布创建及分派事件。

### AC-017

**Given** 工单分类没有有效处理部门，**When** 客服创建工单，**Then** 工单保持 UNASSIGNED，系统生成管理员通知且不伪造分派。

### AC-018

**Given** Agent 已生成工单草稿但客服未确认，**When** 会话继续，**Then** 数据库不存在对应工单，仅保存可审计的草稿建议。

## UC-07 处理工单

### AC-019

**Given** 工单分派给当前部门且状态为 PENDING，**When** 授权人员开始处理，**Then** 工单变为 IN_PROGRESS，版本递增并增加处理记录。

### AC-020

**Given** 员工不属于责任部门且未被直接分派，**When** 尝试处理工单，**Then** API 返回 403，工单状态、版本和记录数量不变。

### AC-021

**Given** 客户端使用旧版本或请求非法状态跳转，**When** 提交处理结果，**Then** API 返回 409，不发生部分更新，并记录冲突指标。

## UC-08 确认与评价

### AC-022

**Given** 本人关联工单为 COMPLETED，**When** 消费者确认结果，**Then** 工单变为 CLOSED，保存关闭记录并显示最终对外结果。

### AC-023

**Given** 工单不是 COMPLETED，**When** 消费者尝试确认，**Then** API 返回 409，状态保持不变且不创建评价。

### AC-024

**Given** 消费者已提交评价，**When** 使用相同业务键再次提交，**Then** 返回原评价且只有一条评价记录，统计事件不会重复计数。
