# 领域术语表

本文档规定项目内的唯一业务术语。数据库、API、代码、界面和文档不得为同一概念使用不同含义。

| 中文术语 | 英文术语 | 唯一定义 |
|---|---|---|
| 消费者 | Consumer | 购买或持有演示订单，并发起咨询、售后申请和评价的 C 端用户。 |
| 人工客服 | Customer Service Agent | 接入消费者会话、回复问题并在需要跨部门处理时创建工单的 B 端人员。 |
| 会话 | Conversation | 一个消费者与平台围绕售后问题进行沟通的有状态容器。一次会话包含多条消息。 |
| 消息 | Message | 会话中不可变的一次发送记录，包含发送方、内容、类型、时间和关联 ID。 |
| 转人工 | Human Handoff | 将由 Mock/AI 处理的活动会话放入人工客服队列的受审计操作。 |
| 客服队列项 | Service Queue Entry | 表示某个会话正在等待客服接入或已被特定客服占用的并发控制记录。 |
| 售后申请 | After-sales Application | 消费者针对订单项提出的退款、退货、换货或质量问题处理请求，属于面向消费者的业务记录。 |
| 售后凭证 | After-sales Evidence | 消费者为售后申请上传的图片或文档元数据及其受控对象存储引用。 |
| 售后审核 | After-sales Review | 授权人员对售后申请作出的批准、驳回或补充材料决定及其不可变记录。 |
| 工单 | Work Order | 需要客服之外的部门继续处理时创建的内部协作与执行记录，不等同于消费者售后申请。 |
| 工单处理记录 | Work-order Record | 工单的一次状态变化、备注、处理动作或系统事件形成的不可变历史。 |
| 工单分派 | Ticket Assignment | 工单当前责任部门和责任人员的关系，以及分派、转派历史。 |
| 工单分类 | Work-order Category | 决定工单处理部门、默认优先级和 SLA 策略的配置。 |
| SLA | Service Level Agreement | 根据工单分类和优先级计算的首次响应或处理完成时限规则。 |
| 知识文章 | Knowledge Article | 经管理和发布、可被搜索或作为 Agent 回答依据的业务内容。 |
| 知识文档 | Knowledge Document | 上传至对象存储并经过解析、分块和索引的原始文件及其元数据。 |
| 租户 | Tenant | 企业数据隔离边界。MVP 运行时只有一个有效企业租户，但租户上下文仍参与数据和权限校验。 |
| Agent 工具 | Agent Tool | Python Agent 可调用的、由 Java 提供并执行权限及业务校验的受控接口。 |
| Mock Agent | Mock Agent | 按固定规则返回确定性结果的 Java 适配器，用于在真实 Python Agent 开发前完成业务闭环和测试。 |
| 关联 ID | Correlation ID | 串联 HTTP 请求、异步事件、Agent 调用、日志和审计记录的全链路标识。 |

## 必须保持的概念边界

1. 售后申请是消费者可见、与订单项关联的业务申请；工单是内部部门协同记录。二者可以关联，但不能合并为同一实体。
2. 会话负责沟通上下文；工单负责后续执行。关闭会话不自动关闭工单。
3. 工单分派表示当前责任关系；工单处理记录保存不可变历史。
4. Agent 只提出回答或动作建议。所有订单、售后和工单变更仍由 Java 业务服务授权并执行。
