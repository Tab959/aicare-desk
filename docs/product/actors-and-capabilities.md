# 角色与业务能力

## 1. 消费者

- 输入：本人身份、本人订单、咨询内容、售后原因、上传凭证和评价。
- 允许：浏览商品；查看本人订单与物流；创建本人会话和售后申请；查看关联工单的对外进度；补充材料；确认结果；评价服务。
- 输出：会话消息、售后申请、凭证、确认和满意度评价。
- 禁止：读取他人订单、会话或售后记录；查看内部工单备注；改变工单分派和内部状态。

## 2. 人工客服

- 输入：等待队列、已接入会话、消费者授权范围内的订单摘要和 Agent 建议。
- 允许：接入队列会话；回复消费者；标记问题类型；发起转派；创建工单；查看本人已处理会话。
- 输出：客服消息、会话总结、工单草稿、工单创建记录和转派记录。
- 禁止：审批财务退款；修改知识库发布状态；查看未接入且无业务关系的完整消费者信息。

## 3. 售后人员

- 输入：分派至售后部门或本人的售后申请与工单。
- 允许：审核售后资格；请求补充材料；选择退货、换货或补偿建议；更新本人负责的工单。
- 输出：售后审核记录、处理意见、工单处理记录。
- 禁止：执行真实资金操作；处理未分派到售后部门的工单。

## 4. 仓库人员

- 输入：分派至仓库部门或本人的退件、换货、补发工单。
- 允许：确认收货；登记质检结果；登记补发或换货出库结果；更新负责工单。
- 输出：仓库处理记录和工单状态变化。
- 禁止：修改退款金额；读取无关消费者会话全文。

## 5. 物流人员

- 输入：分派至物流部门或本人的物流异常工单。
- 允许：登记核查信息；更新物流异常处理结果；建议升级或补发。
- 输出：物流核查记录、升级事件和工单状态变化。
- 禁止：修改原始订单支付信息或审批退款。

## 6. 财务人员

- 输入：分派至财务部门或本人的退款、发票工单及已批准业务结果。
- 允许：确认退款处理状态；登记发票处理结果；更新负责工单。
- 输出：财务处理记录和对消费者可见的结果摘要。
- 禁止：绕过售后审核改变售后决定；访问未分派任务。

## 7. 管理员

- 输入：组织、人员、角色、工单分类、SLA、知识库和 Agent 配置。
- 允许：管理租户内组织与账号；分配角色；维护分类和 SLA；发布知识内容；查看审计与汇总指标。
- 输出：配置版本、权限变更、知识发布记录和审计事件。
- 禁止：静默冒充消费者；删除不可变审计、消息或工单处理历史；绕过资源级权限读取业务内容。

## 8. Agent

- 输入：经 Java 授权后的会话上下文、知识检索结果和业务工具响应。
- 允许：生成答复；建议转人工；生成会话摘要；生成工单草稿；推荐客服回复；调用白名单 Java 工具。
- 输出：带引用的答复、意图、建议动作、置信信号、模型元数据和耗时。
- 禁止：直连业务数据库；持有数据库凭据；自行改变订单、退款或工单；调用未授权工具；隐藏工具调用审计。

## 9. 共同约束

1. 所有角色操作都绑定 `tenant_id` 和经过认证的主体。
2. 部门人员只能处理分派给本人或本部门的工单。
3. 查看权限不自动授予修改权限；状态机和当前分派关系必须同时允许操作。
4. 消费者只能看到对外状态和脱敏结果，不能看到内部备注、风控信息和人员隐私。
5. Agent 的能力不构成独立角色授权，最终授权由 Java 根据委托主体重新判断。
