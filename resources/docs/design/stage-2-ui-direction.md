# Stage 2 UI Direction

## Goal

Stage 2 的页面方向从“通用电商售后中心”调整为“游戏数字商品商城 + AI 客服与工单后台”。页面需要体现游戏消费产品的沉浸感，同时保留企业级后台的清晰权限、工单和审计结构。

## External Inspiration

参考方向来自 Steam、Epic Games Store、PlayStation Store 和 Xbox/Microsoft Store 的公开页面结构：

- 顶部全局导航、搜索、购物车/愿望单/个人入口。
- 首页使用强视觉 Hero、游戏封面卡片、分类筛选、折扣和推荐区域。
- 游戏详情页突出媒体图、版本、标签、平台、价格、购买/礼物动作。
- 支持入口不是独立割裂页面，而是和订单、账号、权益状态关联。

只借鉴信息架构和交互经验，不复制任何品牌资产、图形、商标、文案或具体版式。

## Visual Direction

- Style: premium dark gaming marketplace + cinematic admin console。
- Palette: 深夜蓝黑背景、紫色主品牌、青蓝信息、玫红风险、绿色成功、金色促销。
- Typography: C 端标题更有游戏感，B 端数据使用清晰无衬线和 tabular numbers。
- Shape: 16px 左右卡片圆角，使用玻璃质感边框、轻量 glow 和渐变封面。
- Motion: hover/press 150-240ms；避免重 WebGL；尊重 `prefers-reduced-motion`。
- Accessibility: 深色模式必须保持文本对比；霓虹色只做强调，不承载唯一状态含义。

## Customer Web

C 端信息架构：

- 顶栏：首页、分类、优惠、限时秒杀、订单、客服、公告、搜索、购物车、个人入口。
- 首页 Hero：主推游戏、价格、折扣、平台标签、加入购物车、赠送好友。
- 搜索页：承接首页搜索框，只搜索游戏商品，展示关键词、筛选、排序和搜索结果。
- 搜索与分类：动作、RPG、策略、独立、多人、DLC、CDK、礼物；分类页暂时只保留入口。
- 游戏卡片：封面、标题、标签、平台、价格、折扣、库存/交付方式。
- 游戏详情：媒体墙、版本选择、系统/平台信息、CDK/礼物交付说明、退款资格、客服入口。
- 售后入口：订单问题、CDK 未收到、兑换失败、礼物领取失败、退款申请、账号异常。

## Staff Web

B 端信息架构按角色动态显示：

- 客服：待接待会话、已接待会话、会话详情、用户/订单/权益上下文、Agent 建议、知识命中、升级工单。
- 管理员：会话监控、待处理工单、已处理工单、工单详情、退款审核、CDK/礼物补发、账号处置、客服管理、知识库、运营看板。
- 超级管理员：角色配置、模块权限、SLA 配置、Agent 配置、系统审计，外加管理员全部能力。

后台视觉需要比普通表格后台更高级，但不能牺牲可读性：

- 深色控制台背景。
- 玻璃质感面板。
- 左侧角色感知导航。
- 高风险操作独立分区并使用二次确认语义。
- 权限矩阵和模块可见性使用清晰卡片/表格展示。

## Current Stage 2 UI Acceptance

- customer-web 能静态演示：游戏首页、搜索页、游戏卡片、游戏详情、购物车/个人/客服入口。
- staff-web 能静态演示：客服、管理员、超级管理员三种角色下模块显示差异。
- staff-web 能体现：客服会话分为待接待和已接待；工单分为待处理和已处理；详情页保留上下文、状态流转和审计记录。
- staff-web 能体现：客服只对话和升级，管理员处理工单/退款/账号，超级管理员配置模块权限。
- 390px 移动端不出现横向溢出；B 端表格在移动端用横向滚动或卡片化。
- 所有可点击元素有可见 hover/focus 状态，不使用 emoji 作为结构图标。
