# 游戏数字商品商城与 AI 客服工单平台开发概要

## 1. 项目名称

暂定展示名：**GameCare Desk**

仓库名仍可保持 `aicare-desk`，因为仓库名不必等同最终产品名。

## 2. 项目定位

本项目是一个面向游戏数字商品平台的 AI 客服与工单系统。C 端提供类 Steam/Epic/PlayStation Store/Xbox Store 的游戏浏览、搜索、详情、购物车入口、个人入口和客服入口；B 端提供客服接待、管理员工单处理、退款审核、CDK/礼物补发、账号处置、知识库、数据看板和角色模块权限配置。

项目不做淘宝、拼多多、京东式多商户电商平台，也不做真实支付和真实游戏平台接口。第一版使用虚拟演示数据，重点展示完整业务闭环、工程质量、权限边界、Agent 扩展和可部署能力。

## 3. 核心业务场景

游戏数字商品售后不同于实物电商，第一版聚焦以下问题：

| 场景 | 用户问题示例 | 系统处理方向 |
|---|---|---|
| CDK 未收到 | 我买了游戏但没有收到激活码 | 查询订单和交付状态，必要时升级补发工单 |
| CDK 兑换失败 | 激活码提示已被使用或区域不匹配 | 查询 CDK 状态、平台和区域，升级管理员核查 |
| 礼物领取失败 | 我送给朋友的游戏领取不了 | 查询礼物链接、领取状态、有效期和收礼人信息 |
| 退款申请 | 游戏买错了，可以退款吗 | 根据激活/领取/时限规则生成退款审核工单 |
| 账号异常 | 我的账号被冻结/想注销账号 | 客服沟通原因，管理员处理账号处置工单 |
| 游戏内容咨询 | 这个版本包含哪些 DLC | Agent/知识库直接答复或引导到详情页 |

## 4. 角色设计

| 角色 | 端 | 核心职责 |
|---|---|---|
| 消费者 | C 端 | 浏览游戏、查看详情、查看订单/权益、咨询问题、查看工单进度、评价 |
| 客服 | B 端 | 接入会话、回复用户、查看上下文、采纳 Agent 建议、升级工单 |
| 管理员 | B 端 | 查看全部客服会话、管理客服、处理退款/补发/账号处置工单、维护知识库 |
| 超级管理员 | B 端 | 配置角色可见模块、权限策略、SLA、Agent 配置、系统审计 |
| Agent | 服务端 | 意图识别、知识检索、建议回复、工单草稿、摘要和风险建议 |

客服不直接处理退款、补发或账号处置；这些动作由管理员或超级管理员执行，并写入审计。

## 5. C 端页面规划

第一版 C 端应包含：

- 首页顶部导航：商店、分类、特惠、客服、游戏库、搜索、购物车、个人信息。
- 游戏首页 Hero：主推游戏、价格、折扣、平台标签、购买/赠送动作。
- 游戏分类与搜索：动作、RPG、策略、独立、多人、DLC、CDK、礼物。
- 游戏卡片列表：封面、标题、标签、平台、价格、折扣、交付方式。
- 游戏详情页：媒体图、游戏详情、版本选择、平台信息、系统/区域提示、CDK/礼物交付说明、退款规则、客服入口。
- 订单/权益帮助入口：CDK 未收到、兑换失败、礼物领取失败、退款申请、账号异常。

## 6. B 端页面规划

第一版 B 端应按角色动态显示模块：

### 客服

- 接待队列
- 当前会话
- 用户/订单/权益上下文
- Agent 建议
- 知识库命中
- 升级工单

### 管理员

- 会话监控
- 工单中心
- 退款审核
- CDK/礼物补发
- 账号处置
- 客服管理
- 知识库
- 运营看板

### 超级管理员

- 管理员全部模块
- 角色模块配置
- SLA 配置
- Agent 配置
- 系统审计

## 7. 核心闭环

```text
消费者浏览游戏/查看订单权益
        ↓
发起 CDK、礼物、退款或账号问题咨询
        ↓
Java 校验身份、订单和权益归属
        ↓
Mock Agent 识别意图并给出建议
        ↓
能直接答复？ ── 是 → 返回答复并记录会话
        ↓ 否
转人工客服
        ↓
客服接入、沟通、查看上下文和 Agent 建议
        ↓
客服升级业务工单
        ↓
管理员处理退款、补发或账号处置
        ↓
消费者查看结果、确认并评价
        ↓
审计、统计和知识库沉淀
```

## 8. 技术边界

- 浏览器只调用 Java `platform-api`，不直连 Agent。
- Agent 不访问 MySQL，不持有数据库凭据。
- Java 负责认证、授权、状态机、事务、审计和高风险动作确认。
- Agent 只生成答复、摘要、草稿和建议。
- 模块可见性由后端返回，前端据此显示菜单；动作授权仍由后端强制执行。

## 9. 第一版不做

- 真实支付、真实退款打款。
- 真实 Steam/Epic/PlayStation/Xbox 接口。
- 真实 CDK 采购和库存资金链路。
- 多商户平台、商家后台、分账结算。
- 真实生产多节点高可用。
