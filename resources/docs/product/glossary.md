# 领域术语表

本文档规定项目内的唯一业务术语。数据库、API、代码、界面和文档不得为同一概念使用不同含义。

| 中文术语 | 英文术语 | 唯一定义 |
|---|---|---|
| 消费者 | Consumer | 购买或持有演示游戏数字商品，并发起咨询、工单补充和评价的 C 端用户。 |
| 游戏商品 | Game Product | 平台展示和销售的 Steam 游戏本体。 |
| 游戏版本 | Game Version | 某个游戏商品下的具体版本，例如标准版、豪华版、年度版或 DLC 合集。 |
| 购买方式 | Purchase Method | 平台支持的售卖/交付方式，例如 CDK、Steam 礼物、成品账号、离线账号游玩、授权下载资料。 |
| 可售方案 | Game Offer | 某个游戏版本与某种购买方式的组合，承载价格、库存、状态和交付处理器。 |
| 订单 | Order | 消费者在演示平台购买数字商品形成的交易记录。MVP 不接入真实支付。 |
| 数字权益 | Entitlement | 用户购买后应获得的可使用权益，例如游戏库资格、DLC 资格、CDK 或礼物领取权。 |
| CDK | Redemption Code | 可被用户兑换的游戏激活码或兑换码。 |
| 礼物码 | Gift Code | 可发送给他人领取的数字商品领取凭证。 |
| 权益交付 | Entitlement Fulfillment | 将 CDK、礼物码或游戏库资格交付给用户的过程。 |
| 会话 | Conversation | 消费者与 AI/客服围绕订单、CDK、礼物、退款或账号问题进行沟通的有状态容器。 |
| 转人工 | Human Handoff | 将由 Mock/AI 处理的活动会话放入客服队列的受审计操作。 |
| 客服队列项 | Service Queue Entry | 表示会话正在等待客服接入或已被特定客服占用的并发控制记录。 |
| 业务工单 | Work Order | 需要管理员处理的内部业务任务，例如退款审核、CDK 补发、礼物重发、账号冻结/解冻/注销审核。 |
| 工单处理记录 | Work-order Record | 工单的一次状态变化、备注、处理动作或系统事件形成的不可变历史。 |
| 工单分类 | Work-order Category | 决定处理规则、默认优先级、SLA 和可执行动作的配置。 |
| 退款审核 | Refund Review | 管理员根据数字商品激活、领取、时限和风控规则决定是否批准退款。 |
| 账号处置 | Account Action | 对用户账号执行冻结、解冻、注销审核、风险标记等受控动作。 |
| 模块权限 | Module Permission | 控制某角色在 B 端能看到哪些系统模块的配置，不等同于具体动作权限。 |
| SLA | Service Level Agreement | 根据工单分类和优先级计算的首次响应或处理完成时限规则。 |
| 知识文章 | Knowledge Article | 经管理和发布、可被搜索或作为 Agent 回答依据的业务内容。 |
| 租户 | Tenant | 企业数据隔离边界。MVP 只有一个有效平台租户，但租户上下文仍参与数据和权限校验。 |
| Agent 工具 | Agent Tool | Python Agent 可调用的、由 Java 提供并执行权限及业务校验的受控接口。 |
| Mock Agent | Mock Agent | 按固定规则返回确定性结果的 Java 适配器，用于在真实 Python Agent 开发前完成业务闭环和测试。 |
| 关联 ID | Correlation ID | 串联 HTTP 请求、异步事件、Agent 调用、日志和审计记录的全链路标识。 |

## 必须保持的概念边界

1. 会话负责沟通上下文；工单负责管理员后续业务处理。关闭会话不自动关闭工单。
2. 客服只负责沟通和升级；退款、补发、账号处置由管理员或超级管理员执行。
3. 模块可见性是前端导航和页面入口控制；动作授权仍以后端权限、状态机和资源规则为准。
4. Agent 只提出回答或动作建议。所有订单、权益、退款、账号和工单变更仍由 Java 业务服务授权并执行。
