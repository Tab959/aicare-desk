# MVP 验收标准

以下标准是第 1 至第 6 阶段自动化测试和最终演示的追踪源。

## UC-01 游戏浏览与详情

### AC-001

**Given** 平台存在演示游戏数据，**When** 用户打开 customer-web，**Then** 页面展示顶部导航、搜索、分类、Hero、游戏卡片、购物车入口和个人入口。

### AC-002

**Given** 用户点击任一游戏卡片，**When** 进入详情视图，**Then** 页面展示媒体图、版本、平台、标签、价格、CDK/礼物交付说明、退款规则和客服入口。

### AC-003

**Given** 搜索关键词没有结果，**When** 用户提交搜索，**Then** 页面展示建议分类和热门游戏，不出现空白死路。

## UC-02 订单/CDK/礼物咨询

### AC-004

**Given** 已登录消费者拥有订单 A，**When** 从订单 A 或权益 A 发起咨询，**Then** API 返回 201 和会话 ID，数据库保存绑定订单/权益、消费者和租户的活动会话。

### AC-005

**Given** 消费者不拥有权益 B，**When** 尝试从权益 B 发起咨询，**Then** API 返回资源不可用响应，且不创建会话、不泄露权益 B 的业务字段。

### AC-006

**Given** 同一消费者对订单 A 使用同一幂等键连续提交两次，**When** 两个请求完成，**Then** 返回相同会话 ID，数据库只有一个活动会话。

## UC-03 Mock Agent 回答

### AC-007

**Given** 会话为 BOT_ACTIVE 且 CDK 状态为待交付，**When** 消费者询问“CDK 没收到”，**Then** Mock Agent 返回确定性待交付答复，消费者消息和答复各持久化一次。

### AC-008

**Given** 会话为 BOT_ACTIVE 且礼物领取失败，**When** 消费者询问“朋友领不了礼物”，**Then** Mock Agent 返回礼物状态、可能原因和转人工建议。

### AC-009

**Given** 远程 Agent 超时，**When** AgentGateway 达到超时阈值，**Then** Java 返回降级答复、记录依赖失败和关联 ID，不丢失消费者消息。

## UC-04 转人工

### AC-010

**Given** BOT_ACTIVE 会话，**When** 消费者请求转人工，**Then** 会话变为 WAITING_HUMAN，创建一个活动队列项并发布转人工事件。

### AC-011

**Given** 会话已经 WAITING_HUMAN，**When** 再次请求转人工，**Then** API 返回现有队列状态，活动队列项数量仍为一。

### AC-012

**Given** CLOSED 会话，**When** 请求转人工，**Then** API 返回 409 和稳定业务错误码，不创建队列项。

## UC-05 客服接入与升级

### AC-013

**Given** 两名有权限客服同时接入同一等待会话，**When** 并发请求完成，**Then** 只有一人获得会话，另一请求返回 409，当前分派保持唯一。

### AC-014

**Given** 客服已接入，**When** 客服回复用户，**Then** 消息持久化并能按游标补拉。

### AC-015

**Given** 客服正在处理会话，**When** 客服提交退款或账号问题工单，**Then** 系统创建 PENDING 工单并分派到管理员队列；客服不能直接执行退款或账号处置。

## UC-06 管理员处理工单

### AC-016

**Given** 管理员打开退款工单，**When** 审核通过并提交原因，**Then** 工单写入退款审核记录并迁移到 COMPLETED 或 WAITING_CUSTOMER。

### AC-017

**Given** 管理员打开 CDK/礼物补发工单，**When** 执行补发，**Then** 系统创建新的权益交付记录、废弃旧异常权益引用并写入审计。

### AC-018

**Given** 管理员打开账号处置工单，**When** 执行冻结/解冻/注销审核，**Then** 账号状态变化必须有原因、二次确认和审计事件。

### AC-019

**Given** 无管理员权限的客服，**When** 尝试处理退款、补发或账号处置工单，**Then** API 返回 403，工单状态、版本和记录数量不变。

## UC-07 角色模块配置

### AC-020

**Given** 超级管理员修改客服角色可见模块，**When** 保存配置，**Then** 生成新的配置版本和审计事件。

### AC-021

**Given** 客服重新进入系统，**When** 加载菜单，**Then** 只显示客服被授权的模块，不显示退款审核、账号处置和权限配置入口。

### AC-022

**Given** 超级管理员试图移除自身权限配置模块，**When** 提交保存，**Then** API 返回 409，保留原配置。

## UC-08 确认与评价

### AC-023

**Given** 本人关联工单为 COMPLETED，**When** 消费者确认结果，**Then** 工单变为 CLOSED，保存关闭记录并显示最终对外结果。

### AC-024

**Given** 消费者已提交评价，**When** 使用相同业务键再次提交，**Then** 返回原评价且只有一条评价记录，统计事件不会重复计数。
