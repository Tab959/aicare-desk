# 角色与业务能力

## 1. 消费者

- 输入：本人身份、游戏搜索条件、订单、CDK/礼物问题、咨询内容、凭证和评价。
- 允许：浏览游戏；查看本人订单、CDK、礼物和权益；创建本人会话；查看关联工单对外进度；补充材料；确认结果；评价服务。
- 输出：会话消息、工单补充材料、确认结果和满意度评价。
- 禁止：读取他人订单、CDK、礼物、会话或内部工单备注；执行退款、账号处置或权限操作。

## 2. 客服

- 输入：等待队列、已接入会话、消费者授权范围内的订单/权益摘要、Agent 建议和知识库命中。
- 允许：接入队列会话；回复消费者；标记问题类型；查看本人处理会话；将问题升级为业务工单草稿或提交工单。
- 输出：客服消息、会话摘要、工单草稿、升级原因和转交记录。
- 禁止：直接退款；直接补发 CDK；冻结、解冻、注销用户账号；配置角色权限；删除审计或会话历史。

## 3. 管理员

- 输入：客服会话、业务工单、订单/权益上下文、退款规则、账号风险信息、客服绩效和审计摘要。
- 允许：查看所有客服会话；管理客服账号与排班状态；处理业务工单；审核退款；执行受控 CDK/礼物补发；处理账号冻结、解冻和注销审核；维护知识库；查看运营看板。
- 输出：工单处理记录、退款审核记录、权益补发记录、账号处置记录、客服管理记录和审计事件。
- 禁止：配置超级管理员权限；静默删除业务历史；绕过状态机和二次确认执行高风险动作。

## 4. 超级管理员

- 输入：角色、模块、权限、SLA、系统配置、审计和全量运营数据。
- 允许：拥有管理员能力；配置角色可见模块；配置权限策略；配置 SLA；管理知识库发布；查看完整审计；管理系统级参数。
- 输出：角色配置版本、模块授权变更、权限策略变更、系统配置变更和审计事件。
- 禁止：物理删除不可变审计、聊天消息、订单权益和工单处理历史。

## 5. Agent

- 输入：经 Java 授权后的会话上下文、订单/权益摘要、知识检索结果和业务工具响应。
- 允许：生成答复；建议转人工；生成会话摘要；生成工单草稿；推荐客服回复；推荐退款/补发/账号处置方向；调用白名单 Java 工具查询上下文。
- 输出：带来源的答复、意图、建议动作、风险等级、置信信号、模型元数据和耗时。
- 禁止：直连业务数据库；持有数据库凭据；自行退款、补发、封禁、注销账号或关闭工单；调用未授权工具；隐藏工具调用审计。

## 6. 角色与模块显示原则

系统前端菜单由“角色 + 模块授权 + 资源级权限”共同决定：

- 客服默认显示：接待队列、当前会话、知识库提示、我的会话记录。
- 管理员默认显示：会话监控、工单中心、退款审核、账号处置、客服管理、知识库、运营看板。
- 超级管理员默认显示：管理员全部模块、角色配置、模块权限、系统审计、SLA 配置、Agent 配置。

模块可见不等于动作授权。高风险动作仍必须通过 Java 后端的 RBAC、资源归属、状态机、幂等和审计校验。
