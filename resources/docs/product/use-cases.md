# 核心业务用例

## UC-01 消费者浏览游戏并进入详情

**前置条件：** 平台存在演示游戏、DLC、CDK 或礼物商品。

**触发：** 消费者打开首页、使用搜索或点击分类/游戏卡片。

**主流程：**

1. customer-web 展示游戏首页、分类、搜索、推荐和促销卡片。
2. 消费者点击游戏卡片进入详情视图。
3. 详情页展示媒体图、版本、标签、平台、价格、交付方式、退款规则和客服入口。

**替代与失败：**

- 搜索无结果时展示建议分类和热门游戏，不显示空白死路。
- 商品下架时详情页显示不可购买但保留客服和订单帮助入口。

**可观察结果：** 用户能从首页进入详情，并明确看到购买、赠送和客服入口。

## UC-02 消费者围绕订单/CDK/礼物发起咨询

**前置条件：** 消费者已登录，订单或数字权益属于本人。

**触发：** 消费者在订单、游戏详情或客服入口选择“咨询此订单/权益”。

**主流程：**

1. Java 校验租户、消费者身份、订单/权益归属。
2. 系统创建或复用活动会话。
3. customer-web 打开会话并展示脱敏订单、CDK/礼物和权益摘要。
4. 系统记录会话创建审计和关联 ID。

**替代与失败：**

- 登录过期返回 401，前端保留上下文并引导重新登录。
- 资源不属于当前消费者时返回资源不可用，不泄露资源是否存在。
- 重复点击使用幂等键，只产生一个活动会话。

**可观察结果：** 返回会话 ID；数据库存在唯一活动会话；消费者只能看到本人订单/权益摘要。

## UC-03 Mock Agent 回答游戏数字商品问题

**前置条件：** 会话处于 BOT_ACTIVE，订单/权益上下文有效。

**触发：** 消费者发送“CDK 没收到”“礼物领取失败”“能退款吗”等问题。

**主流程：**

1. Java 保存消费者消息。
2. agent-gateway 调用 Mock Agent。
3. Mock Agent 根据订单、CDK、礼物、退款状态返回确定性答复、问题类型和建议动作。
4. Java 保存答复并推送给消费者。

**替代与失败：**

- Mock 规则不命中时，返回无法自动解决的标准答复并提供转人工操作。
- AgentGateway 超时或真实 Agent 不可用时，走确定性降级，不阻塞会话。
- 重复消息事件通过消息幂等键只入库一次。

**可观察结果：** 会话中出现消费者消息和系统答复；关联 ID 一致；无模型密钥也能完成。

## UC-04 会话转入客服队列

**前置条件：** 会话处于 BOT_ACTIVE，尚未被人工客服接入。

**触发：** 消费者主动转人工，或 Mock/AI 判断需要客服沟通。

**主流程：**

1. Java 校验当前会话状态。
2. 会话迁移到 WAITING_HUMAN。
3. 系统创建唯一客服队列项。
4. staff-web 的等待队列收到更新。

**替代与失败：**

- 重复转人工请求返回现有队列状态，不重复建队列项。
- 已关闭会话拒绝转人工并返回 409。
- 推送失败不回滚队列状态，由客户端轮询或重连恢复。

**可观察结果：** 唯一活动队列项、状态变化记录和 HumanHandoffRequested 事件。

## UC-05 客服接入、回复并升级业务工单

**前置条件：** 客服已登录并具有 conversation:accept；会话正在等待人工。

**触发：** 客服点击“接入”并判断问题无法只靠对话解决。

**主流程：**

1. Java 以乐观锁或原子操作占用队列项。
2. 会话迁移到 HUMAN_ACTIVE 并绑定客服。
3. staff-web 展示消费者、订单、CDK/礼物、历史会话和 Agent 建议。
4. 客服回复用户，必要时提交业务工单草稿。
5. 工单进入管理员待处理队列。

**替代与失败：**

- 两名客服同时接入时，仅一人成功；另一人收到 409。
- 客服无权执行退款/账号处置，只能升级工单。
- WebSocket 断开后消息仍持久化，重连按游标补拉。

**可观察结果：** 会话只有一个当前客服；客服只能沟通和升级；接入、回复和升级均有审计。

## UC-06 管理员处理业务工单

**前置条件：** 管理员已登录，工单状态允许处理。

**触发：** 管理员打开工单中心并处理退款、补发或账号处置任务。

**主流程：**

1. Java 校验管理员角色、模块权限、资源范围和当前版本。
2. 工单从 PENDING 迁移到 IN_PROGRESS。
3. 管理员选择处理动作：退款审核、CDK 补发、礼物重发、账号冻结/解冻/注销审核、规则驳回。
4. 高风险动作要求填写原因并写入审计。
5. 处理完成后迁移到 COMPLETED 并通知消费者。

**替代与失败：**

- 无权限员工返回 403。
- 版本冲突或非法状态跳转返回 409，不产生部分更新。
- 退款/账号处置二次确认失败时不改变业务状态。

**可观察结果：** 每次变化都有不可变处理记录、操作者、时间、动作原因和事件。

## UC-07 超级管理员配置角色模块

**前置条件：** 超级管理员已登录。

**触发：** 超级管理员进入角色配置或模块权限页面。

**主流程：**

1. 系统展示角色列表、模块列表和当前授权矩阵。
2. 超级管理员修改某角色可见模块。
3. Java 校验权限并保存配置版本。
4. 相关用户下次刷新或重新登录后看到新的菜单和模块入口。
5. 系统写入审计事件。

**替代与失败：**

- 试图移除超级管理员自身关键模块时返回 409。
- 模块可见配置不直接授予高风险动作权限。

**可观察结果：** 不同角色看到不同模块；权限变更有版本和审计。

## UC-08 消费者确认结果并评价

**前置条件：** 关联工单处于 COMPLETED，消费者拥有对应会话或订单/权益。

**触发：** 消费者确认结果并提交 1 至 5 分评价。

**主流程：**

1. Java 校验消费者资源归属和工单状态。
2. 工单迁移到 CLOSED。
3. 会话在无其他活动任务时迁移到 RESOLVED 或 CLOSED。
4. 保存唯一满意度评价并更新统计事件。

**替代与失败：**

- 非本人资源返回资源不可用结果。
- 非 COMPLETED 工单拒绝确认并返回 409。
- 重复评价返回原评价；已提交评价不可覆盖，只允许管理员追加审计备注。

**可观察结果：** 工单关闭、评价持久化、消费者可见最终结果，统计异步更新。
